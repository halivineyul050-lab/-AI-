# 泥壳AI工具站上线前检查清单

## 必须完成

- 准备一个稳定的 Node.js 22 LTS 运行环境和持久化磁盘。
- 配置生产环境变量，不要把 `.env` 提交到 Git：

```text
NODE_ENV=production
HOST=127.0.0.1
PORT=4173
NIKE_DB_PATH=/var/lib/nike-ai/nike.db
NIKE_ADMIN_TOKEN=<至少 32 位随机令牌>
NIKE_ANALYTICS_SALT=<独立的高强度随机盐值>
NIKE_ENABLE_TOKEN_ADMIN=false
NIKE_AUTO_SEED=false
NIKE_ALLOWED_ORIGINS=https://你的正式域名
NIKE_TRUST_PROXY=true
```

`NIKE_TRUST_PROXY=true` 只在 Node 服务前面有 Nginx、Caddy 或云负载均衡，并且反向代理只允许本机访问 Node 端口时使用。没有反向代理时保持 `false`。

- 使用 HTTPS 反向代理，公网不要直接暴露 Node 端口。
- 首次上线前备份 `NIKE_DB_PATH` 对应的数据库，并同时保留同目录下的 `-wal` 与 `-shm` 文件，或在停服后再复制整个数据库目录。
- 确认 DNS、HTTPS 证书、域名回源和防火墙规则已经生效。
- 上线前执行：

```powershell
npm test
npm run logos:verify
```

上线后检查：

```text
GET https://你的正式域名/api/v1/health/live
GET https://你的正式域名/api/v1/health/ready
GET https://你的正式域名/api/v1/site/bootstrap
```

两个健康接口都应返回 HTTP 200，且 `data.status` 为 `ok`。然后实际打开首页、工具详情、官网跳转、投稿表单和后台 `admin.html#cms` 各验证一次。

## 上线前建议处理

- 首页当前依赖 unpkg 的 Lucide 脚本、Unsplash 文章图片和少量 Google favicon 回退。正式发布前建议把脚本和文章封面本地化，避免第三方服务故障导致页面不完整。
- 当前后台使用共享 Bearer Token，适合单管理员 MVP，不等同于生产级账号体系。正式运营前应增加管理员账户、MFA、RBAC、短会话和令牌轮换。
- 为 SQLite 设置每日备份、异地备份和恢复演练；同时监控磁盘空间、WAL 文件大小、5xx 比例和 `/health/ready`。
- 核对所有工具名称、价格、Logo、文章图片授权、推广标识和官网链接，尤其是商业推广内容。
- 配置网站标题、描述、分享图、站点地图和 robots 规则，并在正式域名下重新检查 canonical、外链和移动端页面。

## 当前部署边界

- 当前架构是单机 Node.js + SQLite，适合小流量上线；多实例或高并发前需要迁移到 PostgreSQL，并将 Logo、封面迁移到对象存储。
- 邮件订阅目前只入库，尚未接入正式邮件发送服务。
- 内容后台保存后会直接进入数据库，前端刷新即可读取；带有 `cms_managed_at` 的内容不会被服务重启时的种子同步覆盖。
