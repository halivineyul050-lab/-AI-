# AI工具批量导入与合规说明

## 一、结论

截至 `2026-07-14`，对 [ai-bot.cn](https://ai-bot.cn/) 的公开目录结构进行只读审计后，可以确认：

- Sitemap 中有 `584` 个唯一 `/sites/{id}.html` 工具地址。
- 公开分类与分页可发现约 `1,487` 个唯一工具地址。
- 分类页与 Sitemap 的原始 URL 并集为 `1,582`。
- 另有约 `410` 个 `/app/{id}.html` 独立应用页面。
- 部分旧地址会重定向到新地址，因此最终唯一产品数需要按最终地址和官网进行二次去重。

容量规划应按 `1,500–2,000` 条目录记录设计，不能继续使用一次加载和渲染全部工具的方式。

当前官方独立核验批次已完成 `101` 条并导入泥壳AI工具站；数据库当前为 `130` 条工具记录，其中 `129` 条已发布。后续可继续追加新的官网核验批次，导入器会按来源键、官网地址和名称去重。

## 二、为什么没有直接整站搬运

[robots.txt](https://ai-bot.cn/robots.txt) 明确对以下 AI 爬虫设置了全站禁止：

- `ChatGPT-User`
- `GPTBot`
- `ClaudeBot`
- `anthropic-ai`
- 其他自动采集代理

此外，工具详情正文、站内简介、Logo、截图和文章属于对方站点或第三方品牌的内容资产。泥壳AI工具站不会：

1. 绕过 robots.txt 批量抓取详情页。
2. 复制对方原创介绍、正文和 SEO 文案。
3. 下载并重新发布对方托管的 Logo、截图或广告素材。
4. 访问 robots.txt 禁止的 `/go/`、搜索和后台路径。

完成全量导入需要满足以下任一条件：

- 目标站点提供书面授权和允许使用的导出文件。
- 用户从有权访问的后台导出 CSV、JSON 或 NDJSON。
- 改为从各工具官方网站独立采集、核验并撰写原创内容。

## 三、已实现的导入架构

```text
授权 CSV / JSON / NDJSON
→ 本地格式解析
→ URL 安全校验与规范化
→ 来源分类映射
→ 来源键去重
→ 官网规范地址去重
→ 域名 + 产品名去重
→ review 审核队列
→ 人工核验后发布
```

主要文件：

- `scripts/import-tool-catalog.mjs`：本地授权文件导入命令。
- `backend/tool-import.mjs`：规范化、去重和事务入库。
- `backend/migrations/004_tool_catalog_imports.sql`：来源和导入批次数据结构。
- `backend/catalog/category-mapping-ai-bot.json`：来源分类到泥壳分类的映射。
- `imports/tool-catalog.template.csv`：导入模板。
- `tests/import.test.mjs`：幂等、去重、回滚和安全测试。

导入器只读取本地文件，不包含联网抓取功能。

## 四、导入字段

| 字段 | 必填 | 说明 |
|---|---:|---|
| `sourceKey` | 是 | 来源记录的稳定标识 |
| `sourceDetailUrl` | 是 | 获得授权的来源详情地址 |
| `sourceListingUrl` | 否 | 来源分类或列表地址 |
| `sourceCategory` | 是 | 来源分类名称 |
| `name` | 是 | 工具名称 |
| `officialUrl` | 是 | 工具官方网站 |
| `logoUrl` | 否 | 已获授权或官方允许引用的 Logo 地址 |
| `pricingType` | 否 | `unknown/free/freemium/trial/paid/contact` |
| `language` | 否 | `unknown/zh/multi` |
| `platforms` | 否 | `web/desktop/mobile/api`，用 `|` 分隔 |
| `summary` | 否 | 默认不导入，需明确授权参数 |
| `description` | 否 | 默认不导入，需明确授权参数 |
| `features` | 否 | 独立整理的能力短语，CSV 中使用 `|` 分隔 |
| `useCases` | 否 | 独立整理的使用场景，CSV 中使用 `|` 分隔 |
| `verifiedAt` | 否 | 最近核验日期，格式 `YYYY-MM-DD` |

未知价格和语言会明确保存为 `unknown`，不会伪装成免费或多语言。

## 五、运行方式

先复制并填写模板：

```text
imports/tool-catalog.template.csv
```

演练导入，不写入数据库：

```powershell
npm run catalog:import -- --input imports/my-authorized-tools.csv --provider authorized-export --dry-run
```

导入审核队列：

```powershell
npm run catalog:import -- --input imports/my-authorized-tools.csv --provider authorized-export
```

仅当数据已经人工核验时直接发布：

```powershell
npm run catalog:import -- --input imports/my-authorized-tools.csv --provider authorized-export --publish
```

仅当摘要和详情文案也获得使用授权时：

```powershell
npm run catalog:import -- --input imports/my-authorized-tools.csv --provider authorized-export --accept-editorial-text
```

官方官网独立核验批次的演练与正式导入：

```powershell
npm run catalog:import:official -- --dry-run
npm run catalog:import:official
```

## 六、去重规则

按以下优先级匹配：

1. `provider + sourceKey`。
2. 同一来源的详情地址。
3. 规范化后的官网 URL。
4. 相同官网域名和标准化产品名称。

规范化会移除 URL fragment、默认端口和常见追踪参数，并拒绝本机、私网和非 HTTP(S) 地址。

同域名的不同产品不会仅凭域名自动合并。

## 七、仍需外部输入

要把目标站目录全量入库，请提供以下任一内容：

- 目标站后台导出的 CSV/JSON/NDJSON。
- 目标站允许本项目批量采集和使用目录事实字段的授权说明。
- 一个你拥有使用权的工具目录数据源。

获得合法数据后，可直接使用现有导入器完成去重、审核和入库，无需重新开发采集系统。
